# Roadmap

This roadmap is directional; product stakeholders should set priorities and dates before delivery commitments.

## Phase 1: Platform Foundation

- Complete password hashing, JWT authentication, guards, and role-based authorization.
- Replace frontend mock authentication with the backend token flow and error handling.
- Implement users, employees, and departments.
- Establish migrations, seed data, tests, and CI quality gates.
- Publish OpenAPI documentation from implemented controllers.

## Phase 2: Environmental Accounting

- Implement operation records, emission factors, and carbon transactions.
- Add calculation validation and immutable factor-value snapshots.
- Deliver sustainability goals and departmental emissions views.

## Phase 3: Social Engagement

- Deliver CSR participation, evidence review, challenges, XP, badges, and rewards.
- Add diversity and training metrics with appropriate access control.
- Build approval and participation notifications.

## Phase 4: Governance and Compliance

- Implement policy publishing and acknowledgement reminders.
- Deliver audits, compliance issues, ownership, due dates, and resolution workflows.
- Add audit logs and exportable compliance evidence.

## Phase 5: Insights and Scale

- Provide dashboards, scheduled reports, exports, and configurable ESG score weights.
- Connect placeholder frontend pages to implemented API endpoints with loading, empty, and error states.
- Add data-quality checks, imports, retention policies, observability, and deployment hardening.

## Future Enhancements

- Multi-tenant organisation support.
- SSO with SAML or OpenID Connect.
- Emission-factor provider synchronisation and source provenance.
- File evidence storage with malware scanning and retention controls.
- Configurable disclosure-framework report templates.
- Background jobs for notifications, reports, and recalculations.
